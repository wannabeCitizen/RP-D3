Hackathons

Hackathons take what students learned from a previous week's learning challenge and have them test their skills. Students are presented with a scenerio and are then required to take what they learn to complete the hackathon in a 3 hour timespan.

Week 6 Example: Flask/SQLAlchemy Hackathon

Task

Develop a web server for an “Exotic Pet Breeding” site. Tailor your site to a particular exotic species. Be creative and imaginative in which species your team choose to support.

Server-side Database Specification

Unlike in the last week hackathon where the database functionality is fake, this week’s hackathon requires your team to connect your Flask server to a real database. The database should meet the minimum requirements below: It should have one table storing data about pets. The table should have at least 3 columns. One of the columns should be name. The other two should be based on your team’s imagination. The table should have at least 10 rows (i.e., 10 pets). Submission: After these requirements are met, take a screenshot of the content of the table as seen in a GUI viewer (e.g., SQL Manager)



Server-side API Specifications

The web server should provide two services:

Search and View.

1. Search:
  • Its location is http://localhost:5000/search
  • A usage example is: http://localhost:5000/search?n=1&type=metal
  • It returns an HTML page showing a list of results.
  • It can take a take at least a combination of TWO arguments as search criteria. For examples:
      http://localhost:5000/search would return all results
      http://localhost:5000/search?color=red would return all red pets.
      http://localhost:5000/search?color=red&trait=aggresive would return all red pets who are aggressive.

2. View:
  • Its location is http://localhost:5000/view
  • A usage example is http://localhost:5000/view/ben
  • It returns an HTML page about a pet by a particular name (e.g., ben).

Front End Specifications

Design and implement TWO pages using Bootstrap, one page for Search and the other page for View. The Search page's purpose is for displaying search results. You could include a form on the page, but the form doesn't need to actually work. You can demostrate a search function by typing a URL in a browser's address bar. The View page's purpose is for displaying information about something. For both pages, the content must change dynamically to reflect the arguments given in the irl(e.g., search?n=1 should show only one result, view/ben should show the right pet).

Submission: For each screen, submit FIVE screenshots proviing the page content does change dynamically.

Week 3 - Bootstrap Hackathon

Objective

Create a prototype of a new Robot Dating using Bootstrap. Users of this site are single robots who are looking for love (the Valentine's day is fast approaching. Your prototype should have THREE pages)

  1. Sign up page
  2. Search Form
  3. A detailed view of a selected robot

Requirements

  • Be Creative.Yoor team is encouraged to be creative in inventing anything that makes sense for robot dating.
  • Design sketches must be made first. You can use any tool or technique you have learned so far (e.g. Balsamiq, Google Template, hand drawing). You must present a live demo of your sketches to a judge.
  • Knowledge of Bootstrap components must be demonstrated. You must use as many Bootstrap features as possible and appropriate.
  • Interactivity is not required. Each page only needs to contain the right Bootstrap components in the right layout to show your design. The components do not need to be interactive of functional (e.g. links and buttons do not need to work)
